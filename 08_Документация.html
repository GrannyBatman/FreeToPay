  • Введение

    Система FreetoPay сотрудничает с социальными сетями, играми и доутими проектами, которые предусматривают получение пользователями виртуальных бонусов за реальные деньги. Наша система является альтернативой для получения этих бонусов. Вместо того, чтобы производить электронный платеж, пользователю предлагается выполнить рекламные предложения наших партнеров. Для этого мы предоставляем список из нескольких предложений, отобранных соответственно полу, возрасту, месту проживания и предпочтениям пользователя. Вь брав понравившееся ему предложение, пользователь попадает на сайт рекламодателя, где требуется совершить заданное действие (заре-истрироваться. заполнить опрос, совершить покупку)

    Рекламные предложения в системе FreetoPay разделяются на платные и бесплатные. Бесплатными язляются те предложения, для прохождения которых, пользователю не требуется платить (от пользователя требуется пройти регистрацию на сайте, заполнить анкету). К платным предложениям относятся те. для выполнения которых пользователь должен заплатить рекламодателю (сделать покупку в интернет- магазине. оформить платную подписку).

    Ниже описан механизм интеграции для бесплатных предложений рекламодателя.

  • Регистрационные данные рекламодателя

    Каждый зарегистрированный в системе FreetoPay рекламодатель получает логин и пароль для входа в личную панель управления. В панели управления рекламодатель может регистрировать в системе FreetoPay свои рекламные предложения.



    Для регистрации рекламного предложения, нужно указать следующие данные:

    • Название рекламного предложения (его увидит пользователь на странице выбора предложений)
    • Адрес LandingURL страницы с рекламным предложением, на которую будут перенаправляться пользователи
    • E-mail адрес для связи с работниками рекламодателя в случае возникновения технических неполадок связанных с данным предложением
    • Краткое описание и инструкция для пользователя по выполнению предложения. Несколько строк текста с указанием на то что конкретно должен сделать пользователь на сайте, куда ведет рекламное предложение (заполнить анкету, зарегистрироваться)
    • Стоимость (в рублях) выполнения рекламного предложения. Те это сумма, которую FreetoPay получает за каждого приведенного к рекламодателю пользователя, который успешно выполнит рекламное предложение (т е. сделает все согласно инструкции из предыдущего пункта)
    • Данные о целевой аудитории, на которую направлено предложение Возраст (минимум, максимум) пользователя, пол (мужской, женский, любой) пользователя и список городов, на которые нацелено это предложение

    После регистрации рекламного предложения рекламодатель получит

    • Уникальный идентификатор Offerjd рекламного предложения
    • Секретный код ApiSecretKeyAdv

  • Механизм работы FreetoPay с рекламодателями

    Когда пользователь выбирает ваше рекламное предложение. FreetoPay перенаправляет его на сайт рекламодателя по адресу LandingURL. указывая в параметрах GET запроса fp_uid=FreetoPayUser_id. Где FreetoPayUserJd - уникальный идентификатор пользователя в системе F-eetoPay. Затем пользователь осуществляет переходы по страницам сайта рекламодателя, заполняет различные формы и т.п.
    Если пользователь выполняет все требования рвкламно-о предложения, которое он выбрал, то рекламодатель должен передать информацию об успешном заврешении пользователем рекламного предложения Эта информация должна быть передана с помощью запроса методом GET на адрес http:/.'freetopay ru/adv/ofler_comp!ete php с параметрами, указывающими уникальный идентификатор пользователя FreetoPayUserJd и идентификатор выполненного предложения Offerjd. Более подробно параметры запроса описаны ниже
    Стоит заметить, что для таких рекламных предложений как заполнение опросов, анкет, подписка на рассылку и прочих бесплатных для пользователя предложений такой механизм реализуется достаточно просто, так как после заполнения анкеты или оформления подписки пользователь чаще всего перенаправляется на страницу, подтверждающую правильность введенных данных. При генерации этой страницы можно легко передать запрос на сервер FreetoPay с оповещением о выполнении пользователем рекламного предложения.
    Рассмотрим вариант, когда вы зарегистрировали в качестве рекламного предложения опрос находящийся по адресу http://my_poII.ru /poll.php. Если пользователь, просматривающий список доступных ему рекламных редложений, выбирает ваше, то FreetoPay присзаизает пользователю уникальный идентификатор FreetoPayUserJd равный, например. 100500 и перенаправляет его на вашу страницу по адоесу http://my_poll.ru/'poll.php?fp_uid=100500. Скрипт poll.php должен каким-либо образом соотнести значение fp_uic=100500 и пришедшего пользователя, чтобы в случае успешного выполнения им рекламного предложения оповестить об этом FreetoPay (наиболее простой способ - записать значение fp_uid=100500 пользователю в cookies). Если пользователь отвечает на все вопросы и заполняет нужные формы, то в результате он попадает на вашем сайте на страницу http://my_poll,ru.'poll Jinish.php. То что пользователь попал на эту страницу, означает что он выполнил все предъявленные к нему требования и завершил выполнение рекламного предложения. Значит нужно послать соответствующий запрос серверу FreetoPay. Это может сделать, например, сам скрипт polljinish.php перед генерацией HTML-кода страницы.

  • «Приземление» пользователей на сайте рекламодателя

    FreetoPay направляет пользователей на ваш сайт по указанному LandingURL. добавляя в параметрах GET запроса fp_uid=FreetcPayUseJd Где FreetoPayUserJd — уникальный идентификатор (натурально число) пользователя в системе FreetoPay Если вы указали LandingURL равный 'http://mysite.ru/index.php?par1ner=freetopay. то пользователь будет навправлен по адресу http://mysite.rufincex.php?partner=freetopay&fp_uid=FreetoPayUserJd. Значение FreetoPayUserJd стоит записать пользователю в Cookies, для того чтобы оповестить FreetoPay в случае если он успешно выполнит рекламное предложение

  • Оповещение FreetoPay о выполненном рекламном предложении

    Если у вас появляется инфрмация что пользователь, пришедший к вам от FreetoPay и имеющий идентификатор FreetoPayUserJd, выполнил ваше рекламное предложение, то на сервер FreetoPay должен быть передан соответствующий запрос. Этот запрос выполняется методом GET на адрес http://freetopay ru/adv/offer_comp!ete php с параметрами

    • fp_uid=FreetoPayUserJd&
    • aid=Offer_d&
    • sig=CheckSum

    FreetoPayUserJd — идентификатор пользователя, который был передан при перенаправлении пользователя на Landing URL Offerjd — идентификатор рекламного предложения, которое прошел пользователь. Значение Offerjd для каждого вашего предложения можно найти в панели управления

    Checksum — контрольная сумма MD5 (строка, содержащая 32 символа 0-9 A-F в любом регистре, представляющая шестнадцатиричное число с 32 разрядами). Формируется из строки, содержащей параметры FreetoPayUserJd. Offerjd и секретный ключ ApiSecretKeyAdv, которые разделены символом двоеточия С:').
